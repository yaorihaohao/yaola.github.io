圣杯和双飞翼布局

Posted on 2019-09-16

花点时间整理下圣杯和双飞翼布局

主要是解决左右两边定长 中间自适应的问题。
当然在现在我觉得如果浏览器环境允许的话 使用Flex 最好代码量也少

先写个总结 圣杯/双飞翼
个人感觉还是圣杯语感比较好一些，因为它把左右两边div进行包裹起来写的。

  • 主要考察的问题 margin-left/right在float环境下的影响 margin的负值会进行覆盖前面的dom元素

  • 主要解决的问题

    • 左右两边固定长度，中间的div进行可适应。
    • 中间部分在dom结构上优先 以便于解析
    • 允许三列中的任意一列为最高列
    • 需要用额外的div

圣杯模式
dom结构信息

.container{
padding: 0 100px;
}
.middle {
width: 100%;
}
.left: {
width: 100px;
margin-left: -100%; // 这里margin对应的百分比是自己的父元素 所以这里就是 middle的宽度咯 负数就是将这个dom往左移动 100%的父宽度。会盖住middle
// 然后进行定位处理
position: relative;
left: -100px; //向左移动100px
}
.right: {
width: 100px;
margin-right: -100px; //同理
}

<div>
    <div class="header"></div>
    <div class="container">
        <div class="middle">middle</div>
    </div>
    <div class="left">left</div>
    <div class="right">right</div>
    <div clas="footer"></div>
</div>

css部分 都是左浮动， 我大概写一下主要区别的内容
.container{
float: left;
width: 100%;
}
.middle{
margin: 0 100px;
}
.left{
width: 100px;
margin-left: -100%; //由于父元素就是100%的长度 所以 双飞翼用不到定位的操作
}
.right{
width: 100px;
margin-left: -100px;
}
```
其他具体部分 可以看自己的印象笔记记录内容
