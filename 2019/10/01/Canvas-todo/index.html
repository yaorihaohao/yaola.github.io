睡不着起来看下canvas初步 进行下总结

  2. 解决了什么问题？
    之前都是直接去使用img标签去进行嵌入图像 我希望去动呢 后来出现了audio video标签但是没有图像相关的。
    极倔web页面只能显示静态图片问题。

支持情况
元素 Chrome IE Firefox Safari Opera
Canvas 4.0+ 9.0+ 2.0+ 3.1+ 9.0+
例子分析 如果需要画一个圆

1.img图片的形式去做 替换的时候的成本太大了 并且会生成请求
2.div + css 只能画简单的东西 复杂的话还是太麻烦
3.svg跟图片类似 但是有些区别
4.canvas + js 可以动态去画

svg中每一个元素其实都是dom会直接导致页面的重新绘制，但他是适量图。
canvas 只有在位置变化，体积 变化的时候会导致页面重绘，依赖分辨率。
性能上 canvas数量较多的情况 > svg的，
如果数量少的话 svg也可行 这个看实际的应用场景。
主要就是要帅

开始使用

定义宽跟高的区别 如果不设定初始值的话是 300 * 150

创建顺序
1.创建获取上下文 el.getContext(‘2d’)
2.开始绘制路线 el.beginPath();
2.绘制路径
3.关闭路径 el.closePath();
4.填充颜色/或秒边

arc(x, y ,r, sAngle, eAngle,顺时针还是逆时针) 画圆
stroke() 画边
moveTo() 移动起始点
lineTo() 画直线到某个点 (lineCap/lineJoin/lineWidth/miterLimit)
fillRact(x, y, width, height) 画实心矩形
strokeRect(x, y, width,height) 空心矩形

fillStyle/strokeStyle/shadowColor/shadowBlur/shadowOffsetX/shadowOffsetY 样式
设置渐变
createLinearGradient(x0,y0, x1, y1); 搭配 addColorStop(number, ‘颜色样式进行使用’)

另外图形转化大致与css中差不多
scale()/rotate( 旋转角度 * Math.PI /180)/translate()/transform/setTransform

最重要一点就是
drawImage() 向画布上绘制图像、画布或者视频
drawImage(‘图像/画布/视频’, 剪切的x坐标, 剪切的y坐标, swith被剪切图像的宽度, sheight, x画布上图像方式的位子, y,width, height)

matchall
